Configuration Semaine 1 : Outils — Hébergement — GitHub Pages

Présentation

GitHub Pages est une fonctionnalité de GitHub qui vous permet d’héberger le contenu de votre dépôt GitHub sous forme de site web avec un nom de domaine simple. Cela vous permet de partager et de soumettre vos travaux déployés pour évaluation.

Cette activité fournit les étapes nécessaires pour activer GitHub Pages sur votre dépôt webacademy sur GitHub.

Instructions de l'activité

  1. Connectez-vous à votre compte GitHub et ouvrez votre dépôt webacademy.
  2. Cliquez sur le menu Paramètres.
  3. Dans le menu de gauche, cliquez sur Pages.
  4. Dans le panneau GitHub Pages, sous Build and deployment, configurez les options suivantes :
    • Source : Déployer depuis une branche
    • Branche : sélectionnez main (ou master) et laissez le chemin sur /(root). Puis cliquez sur Enregistrer.

✅ GitHub Pages est maintenant activé pour votre dépôt webacademy à l’adresse suivante :

https://votre-nom-utilisateur-github.github.io/webacademy

Notez que vous devez avoir du contenu sur le site, comme une page d’accueil par défaut nommée index.html, pour que le site s’affiche. La publication peut prendre quelques minutes après activation et chaque mise à jour.
